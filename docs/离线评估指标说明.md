# 离线评估指标说明

## 脚本路径

```text
evaluate/offline_metrics.py
```

该脚本基于 `test_ratings.csv` 对不同推荐结果文件做离线评估。测试集只作为评估目标，不参与画像构建、召回训练或排序训练。

## 输入文件

```text
data/processed/test_ratings.csv
data/features/movie_profile.csv
data/recall/als_recall.csv
data/recall/itemcf_recall.csv
data/recall/merged_recall_candidates.csv
data/rank/ranked_top50.csv
data/rank/ranked_top10_mmr.csv
```

缺失的模型文件会被跳过并写入日志，不会伪造评估结果。

## 输出文件

```text
data/eval/offline_metrics.csv
data/eval/eval_summary.json
```

`offline_metrics.csv` 字段：

```text
model_name
k
precision
recall
ndcg
hit_rate
coverage
diversity
evaluated_users

```

## 测试集相关电影定义

```text
rating >= 4.0
```

```text
test rows: 610

```

## Precision@K

```text
Precision@K = TopK 命中相关电影数 / K
```

## Recall@K

```text
Recall@K = TopK 命中相关电影数 / 用户测试集中相关电影数
```

## NDCG@K

使用二值相关性：

```text
rel = 1 if movie in relevant set else 0
DCG@K = sum(rel_i / log2(i + 1))
NDCG@K = DCG@K / IDCG@K
```

## HitRate@K

```text
HitRate@K = TopK 是否至少命中一个相关电影
```

命中为 1，否则为 0。

## Coverage@K

```text
Coverage@K = 推荐结果中不同电影数量 / 全部电影数量
```

全部电影数量来自 `movie_profile.csv`。

## Diversity@K

基于推荐列表内部电影类型 Jaccard 相似度计算：

```text
Diversity@K = 1 - average_pairwise_genre_similarity
```

推荐列表少于 2 部电影时，Diversity 记为 0。

## 为什么先不加入长尾指标

当前阶段目标是验证标准推荐链路中的召回、排序、重排效果，因此只计算 Precision、Recall、NDCG、HitRate、Coverage 和 Diversity。没有加入 Novelty、LongTailRatio 或长尾电影标记，避免把长尾推荐主题提前混入本阶段结论。
