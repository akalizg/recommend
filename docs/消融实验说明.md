# 消融实验说明

## 脚本路径

```text
evaluate/ablation_eval.py
```

该脚本读取离线评估结果，整理 ALS、ItemCF、多路召回融合、XGBoost 精排和 MMR 重排的消融对比，并自动生成文字结论。

## 输入文件

```text
data/eval/offline_metrics.csv
```

## 输出文件

```text
data/eval/ablation_metrics.csv
data/eval/ablation_summary.md
```

## 消融实验目的

本阶段目标是用同一份 `test_ratings.csv` 评估各模块对最终推荐效果的影响：

```text
召回通道是否有效
多路召回融合是否改善候选覆盖
XGBoost 精排是否改善排序质量
MMR 重排是否提升推荐列表多样性
```

## 对比模型

```text
ALS
ItemCF
ALS+ItemCF_Merged
XGBoost_Top50
XGBoost_MMR_Top10
```

当前阶段不加入 LightGCN，也不加入长尾推荐。

## 多路召回消融

本次 @10：

```text
ALS Recall@10: 0.005525
ALS+ItemCF_Merged Recall@10: 0.041322
```

相较 ALS 单路召回，多路融合 Recall@10 提升 `+0.035797`，说明融合召回增强了候选覆盖能力。不过 ItemCF 单路在本次留一评估中表现更强，说明当前融合权重仍有优化空间。

## XGBoost 精排消融

本次 @10：

```text
ALS+ItemCF_Merged Precision@10: 0.004132
XGBoost_Top50 Precision@10: 0.000275
ALS+ItemCF_Merged NDCG@10: 0.029986
XGBoost_Top50 NDCG@10: 0.000918
```

当前 MovieLens small 留一测试下，XGBoost 精排没有带来 Precision/NDCG 提升。可能原因包括候选集正样本较少、训练目标与留一评估不完全一致、排序特征还比较基础。

## MMR 重排消融

本次 @10：

```text
XGBoost_Top50 Diversity@10: 0.695954
XGBoost_MMR_Top10 Diversity@10: 0.918096
```

相较 XGBoost Top50，MMR 将 Diversity@10 提升 `+0.222142`，说明基于类型相似度的标准 MMR 重排有效增加了推荐列表多样性。

同时：

```text
XGBoost_Top50 Precision@10: 0.000275
XGBoost_MMR_Top10 Precision@10: 0.001377
```

本次 MMR 没有牺牲 Precision@10，反而略有提升。但由于测试集正样本较少，这个结论应谨慎看待。

## 实验结论

1. ItemCF 在当前 MovieLens small 留一评估中是最强的 @10 基线。
2. ALS+ItemCF 融合相比 ALS 单路召回提升了 Recall@10，但低于 ItemCF 单路，融合权重后续可调。
3. XGBoost 精排当前未提升 @10 相关性指标，需要继续优化训练样本、特征和目标。
4. MMR 明显提升 Diversity@10，适合作为标准推荐链路中的多样性重排模块继续保留。

## 当前不足

MovieLens small 数据稀疏，且每个用户测试集中只有最新一条评分，导致离线评估命中稀少。当前结果更适合用于模块对比，不宜直接视为线上效果。

## 下一步优化方向

```text
1. 做留多法或时间窗口评估，降低单条测试样本带来的波动。
2. 调整 ALS+ItemCF 融合权重。
3. 优化 XGBoost 训练样本构造，加入更多稳定交叉特征。
4. 对 MMR 的 lambda_rel 做参数扫描，寻找相关性和多样性平衡点。
5. 后续再考虑推荐理由和前端评估页面。
```
