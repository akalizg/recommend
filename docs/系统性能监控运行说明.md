# 系统性能监控运行说明

## 1. 启动方式

监控服务使用独立的 `docker-compose.monitor.yml`，不会改变默认 `docker-compose.yml` 的启动行为。

### 1.1 Docker 方式

在项目根目录执行：

```powershell
docker compose -f docker-compose.yml -f docker-compose.monitor.yml up -d
```

启动后服务地址：

| 服务 | 地址 | 说明 |
| --- | --- | --- |
| FastAPI | `http://localhost:8000` | 推荐系统后端。 |
| Metrics | `http://localhost:8000/metrics` | Prometheus 指标接口。 |
| Prometheus | `http://localhost:9090` | 指标抓取与查询。 |
| Grafana | `http://localhost:3001` | 监控看板。 |

Grafana 默认账号：

```text
admin / admin
```

### 1.2 本地 Grafana 方式

如果本机已经有 Grafana，例如：

```text
D:\grafana\grafana-13.0.1+security-01
```

可以使用本地启动脚本：

```powershell
.\scripts\start_local_monitoring.ps1
```

该脚本会：

- 使用 `config/prometheus.local.yml` 抓取 `http://127.0.0.1:8000/metrics`。
- 使用本项目的 Grafana provisioning 和 dashboard 配置。
- 将 Grafana 启动在 `http://localhost:3001`。
- 如果系统 PATH 中找得到 `prometheus.exe`，会自动启动本地 Prometheus。

如果 Prometheus 没有加入 PATH，可以手动指定：

```powershell
.\scripts\start_local_monitoring.ps1 -PrometheusExe "D:\prometheus\prometheus.exe"
```

注意：只有 Grafana 不够，Grafana 需要 Prometheus 作为数据源；如果没有 Prometheus，看板可以打开但没有数据。

## 2. Prometheus 验收

打开：

```text
http://localhost:9090/targets
```

验收标准：

- `reciperec-api` target 状态为 `UP`。
- Endpoint 指向 `http://api:8000/metrics`。

可以在 Prometheus 查询页测试：

```promql
reciperec_up
system_cpu_usage_percent
http_requests_total
recommend_source_total
feedback_events_total
```

## 3. Grafana 验收

打开：

```text
http://localhost:3001
```

进入：

```text
Dashboards -> RecipeRec -> RecipeRec System Monitoring
```

看板应包含：

- CPU 使用率。
- 内存使用率。
- 进程内存。
- 磁盘使用率。
- HTTP QPS。
- HTTP 平均耗时和 P95 耗时。
- HTTP 状态码和异常数。
- 推荐来源。
- 推荐缓存命中。
- 推荐空结果和兜底。
- 反馈事件。
- 点击、收藏、评分事件。

## 4. 生成测试数据

访问几个接口后再观察看板：

```powershell
Invoke-WebRequest http://localhost:8000/health
Invoke-WebRequest http://localhost:8000/metrics
Invoke-WebRequest http://localhost:8000/recipes/recommend/1
Invoke-WebRequest "http://localhost:8000/recipes/search?q=chicken"
```

如果需要验证反馈指标，可以调用反馈接口：

```powershell
Invoke-WebRequest `
  -Method POST `
  -Uri http://localhost:8000/feedback `
  -ContentType "application/json" `
  -Body '{"user_id":1,"movie_id":10,"feedback_type":"click","rank_position":1}'
```

## 5. 停止监控

停止所有服务：

```powershell
docker compose -f docker-compose.yml -f docker-compose.monitor.yml down
```

只停止监控服务：

```powershell
docker compose -f docker-compose.yml -f docker-compose.monitor.yml stop prometheus grafana
```

## 6. 文件说明

| 文件 | 说明 |
| --- | --- |
| `config/prometheus.yml` | Prometheus 抓取 FastAPI `/metrics` 配置。 |
| `config/prometheus.local.yml` | 本地 Prometheus 抓取 `127.0.0.1:8000/metrics` 配置。 |
| `config/grafana/provisioning/datasources/prometheus.yml` | Grafana 自动配置 Prometheus 数据源。 |
| `config/grafana/provisioning/dashboards/dashboards.yml` | Grafana 自动加载本地看板配置。 |
| `config/grafana/dashboards/reciperec_monitoring.json` | RecipeRec 系统监控看板。 |
| `docker-compose.monitor.yml` | 可选 Prometheus/Grafana 服务定义。 |
| `scripts/start_local_monitoring.ps1` | 本地 Grafana/Prometheus 启动脚本。 |

## 7. 注意事项

- 主 `docker-compose.yml` 未被修改，默认启动方式不受影响。
- Grafana 使用 `3001` 端口，避免和前端 `3000` 冲突。
- Prometheus 容器通过 Docker 服务名 `api:8000` 抓取指标。
- 如果只单独启动 Prometheus/Grafana，而没有启动 `api` 服务，Prometheus target 会显示 `DOWN`。
