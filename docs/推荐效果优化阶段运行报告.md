# 推荐效果优化阶段运行报告

## 本阶段为什么要优化

上一阶段离线评估发现：

```text
ItemCF 的 Precision@10 / Recall@10 / NDCG@10 高于 XGBoost_Top50
ALS+ItemCF 原始融合低于 ItemCF 单路
MMR 提升了 Diversity，但排序相关性仍弱
```

因此本阶段针对三件事做离线优化：

```text
1. 搜索 ALS + ItemCF 召回融合权重
2. 使用 XGBRanker 优化排序训练目标
3. 扫描 MMR lambda 参数
```

本阶段没有修改 FastAPI、Vue、在线推荐链路，也没有覆盖旧模型和旧结果文件。

## 是否成功提升指标

召回融合：

```text
Precision@10: 0.004132 -> 0.009366
Recall@10:    0.041322 -> 0.093664
NDCG@10:      0.029986 -> 0.045264
Diversity@10: 0.763425 -> 0.788934
```

结论：召回融合优化成功。

MMR：

```text
XGBoost_Ranker_Top50 Diversity@10:     0.768832
XGBoost_Ranker_MMR_Top10 Diversity@10: 0.919420
```

结论：MMR lambda 调优继续显著提升多样性。

XGBoost：

```text
XGBoost_Classifier_Top50 NDCG@10: 0.000918
XGBoost_Ranker_Top50 NDCG@10:     0.000918
ItemCF NDCG@10:                   0.041365
```

结论：XGBRanker 成功训练，但没有解决 XGBoost 不如 ItemCF 的问题。当前瓶颈更可能在候选正样本覆盖、训练目标与留一测试不一致、特征表达不足，而不是模型类型本身。

## 下一步建议

1. 优先保留优化后的召回融合权重，因为它已经超过 ItemCF 单路。
2. 继续诊断 XGBoost 排序弱的问题，重点检查候选集中测试正样本覆盖率。
3. 尝试用候选级训练集训练 Ranker，而不是历史评分全集训练。
4. 做 MMR 与优化融合召回的组合评估，但暂不接入在线链路。
