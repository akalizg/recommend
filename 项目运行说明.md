# 食谱推荐系统运行说明

本文档用于本地演示和答辩前检查，按顺序启动即可。

## 一、启动 Elasticsearch

如果 Elasticsearch 已经安装并解压，进入 ES 目录执行：

```powershell
.\bin\elasticsearch.bat
```

检查 `recipes` 索引：

```powershell
curl.exe http://localhost:9200/_cat/indices/recipes?v
```

预期结果：

```text
recipes 索引存在
docs.count 约 178265
health 为 green 或 yellow
```

如果需要重建索引：

```powershell
.\.venv\Scripts\python.exe scripts\index_recipes_to_es.py --es-url http://localhost:9200 --index recipes --recreate --batch-size 1000
```

## 六、Kafka 反馈链路

Kafka 部署在虚拟机 node1：

```text
Kafka 路径：/export/server/kafka_2.12-2.4.1
Broker 地址：node1:9092
反馈 Topic：recipe_feedback
实时推荐 Topic：recipe_realtime_recommend
```

启动 Kafka：

```bash
cd /export/server/kafka_2.12-2.4.1
bin/zookeeper-server-start.sh -daemon config/zookeeper.properties
bin/kafka-server-start.sh -daemon config/server.properties
```

查看 topic：

```bash
bin/kafka-topics.sh --bootstrap-server node1:9092 --list
```

Windows 本机检查端口：

```powershell
Test-NetConnection node1 -Port 9092
```

后端反馈接口现在会在本地落库、更新 Redis 实时画像后，将反馈事件发送到 Kafka：

```powershell
curl.exe -X POST http://localhost:8000/feedback `
  -H "Content-Type: application/json" `
  -d "{\"user_id\":1533,\"movie_id\":11041,\"feedback_type\":\"like\",\"feedback_value\":5,\"rank_position\":1}"
```

返回中的 `kafka_sent=true` 表示反馈事件已发送到 Kafka。

启动 Kafka 消费端更新 Redis 实时画像，并生成 FAISS 实时推荐缓存：

```powershell
.\.venv\Scripts\python.exe scripts\kafka_feedback_consumer.py --max-events 10
```

消费成功后会写入：

```text
user:recent_feedback:{user_id}
user:realtime_profile:{user_id}
recipe:realtime_rec:user:{user_id}:k:{top_k}
```

后端 `/recipes/recommend/{user_id}` 会优先读取 `recipe:realtime_rec:user:{user_id}:k:{top_k}`，没有实时缓存时再回退到离线推荐结果。

如果只想消费反馈并更新画像，不生成实时推荐缓存，可以执行：

```powershell
.\.venv\Scripts\python.exe scripts\kafka_feedback_consumer.py --disable-realtime-recommend
```

如果要持续消费，去掉 `--max-events`：

```powershell
.\.venv\Scripts\python.exe scripts\kafka_feedback_consumer.py
```

## 七、MinIO 离线产物同步

MinIO 用来同步 Spark 离线计算产物。推荐流程是：虚拟机 Spark 跑完离线任务后上传到 MinIO，Windows 本地后端再从 MinIO 下载最新产物。

当前默认配置：

```text
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=reciperec
MINIO_ARTIFACT_PREFIX=offline/latest
```

Windows 本地检查 MinIO 是否可用：

```powershell
.\.venv\Scripts\python.exe scripts\sync_minio_artifacts.py list
```

上传离线产物到 MinIO：

```powershell
.\.venv\Scripts\python.exe scripts\sync_minio_artifacts.py upload
```

只上传指定产物：

```powershell
.\.venv\Scripts\python.exe scripts\sync_minio_artifacts.py upload --paths data/rank/enhanced/enhanced_ranker_metrics.json
```

从 MinIO 下载最新产物到当前项目目录：

```powershell
.\.venv\Scripts\python.exe scripts\sync_minio_artifacts.py download
```

如果是在虚拟机 node1 上上传，而 MinIO 运行在 Windows 本机，需要在虚拟机执行前设置 Windows 主机 IP：

```bash
export MINIO_ENDPOINT=192.168.88.x:9000
export MINIO_ACCESS_KEY=minioadmin
export MINIO_SECRET_KEY=minioadmin
export MINIO_BUCKET=reciperec
export MINIO_ARTIFACT_PREFIX=offline/latest
python scripts/sync_minio_artifacts.py upload
```

`192.168.88.x` 要替换成 Windows 在虚拟机网段能访问到的 IP。上传成功后，Windows 本地执行 `download` 命令即可同步最新 `data/` 和 `models/` 产物。

## 八、常见问题

### 1. Spark 页面正常，但 Windows 直接连接 Spark 失败

虚拟机 Spark 为 3.1.2，Windows 项目 PySpark 为 3.5.3，跨版本直接提交容易失败。当前推荐方式是：

```text
Windows 负责开发和打包
node1 使用 spark-submit 提交任务
三台虚拟机 worker 执行 Spark 作业
```

### 2. Spark 报 node1.itcast.cn:8020 连接失败

这是因为默认 HDFS 没启动。当前项目验证脚本已经加入：

```bash
--conf spark.eventLog.enabled=false
--conf spark.hadoop.fs.defaultFS=file:///
```

如果手写 spark-submit 命令，也要加上这两个参数。

### 3. 官方测试集 Precision/Recall/NDCG 为 0

这是严格同一 `recipe_id` 命中评估导致的结果，主要暴露当前召回候选对官方测试集正样本覆盖不足。展示项目效果时，应主要使用推荐结果质量评估表和 XGBoost 验证集指标。
